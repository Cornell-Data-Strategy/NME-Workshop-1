Verify Git Installation

Open your terminal and run:

git --version

You should see something like:

git version 2.42.0
If you see a version number, you're good!

Configure Git

Tell Git who you are:

git config --global user.name "Your Name"
git config --global user.email "your.email@cornell.edu"

Verify your config:

git config --list

Install GitHub CLI

Download the installer for your OS from: cli.github.com

Run the installer and follow the prompts.

Add Your Name!

Open README.md and add your name:

# NME-Workshop-1

## Contributors
- [Your Name](https://github.com/your-username)
Save the file! (Cmd+S on Mac, Ctrl+S on Windows)

Stage Your Changes

See what files changed:

git status

You should see README.md in red (modified but not staged).

Add it to staging:

git add README.md

Now git status shows it in green!

Commit Your Changes

Create a commit with a message:

git commit -m "Add [Your Name] to contributors"

Replace [Your Name] with your actual name!

Commit messages should be short and describe WHAT you changed.

Push to GitHub!

Send your changes to GitHub:

git push
That's it! Your changes are now on GitHub!

Go to the repo on GitHub and refresh to see your name!

Good Git Habits

Things that will make your future self (and teammates) happy

  • Commit early and often Small commits are easier to understand, review, and undo.
  • One purpose per commit Don't mix "fix bug" with "refactor everything" in the same commit.
  • Write clear messages Use short, action-style messages like Add validation for signup form.
  • Pull before you start Run git pull at the beginning of a work session to avoid surprises.
  • Don't commit secrets API keys, passwords, and private data should never be in Git.
  • Commit working code Avoid pushing broken main; use branches when you're experimenting.

Environment Files (.env)

Keep secrets out of Git, keep config in one place

  • Store API keys, tokens, and passwords locally
  • Switch settings between dev, staging, and prod
  • Prevent sensitive data from leaking to GitHub
APP_ENV=development
API_KEY=sk-live-********
DATABASE_URL=postgres://user:pass@localhost:5432/app
Rule: Never commit real secrets. Use .env locally.

Use .gitignore

Tell Git what NOT to track

Ignore These

  • .env (secrets)
  • node_modules/ (dependencies)
  • *.log (logs)
  • dist/, build/ (outputs)

Example

# .gitignore
.env
node_modules/
dist/
*.log
.DS_Store
__pycache__/
Best practice: Commit .env.example with fake values so teammates know what to set.

Troubleshooting

"Permission denied"

gh auth login

Re-authenticate with GitHub.

"Failed to push - rejected"

git pull

Then try git push again.

"Command not found: git"

Reinstall Git and restart your terminal.
