What is Version Control?

Think of it like Google Docs history, but for code

  • Track every change ever made
  • Go back in time if something breaks
  • Collaborate without overwriting each other's work
  • Work in parallel on different features

Let's Get Started!

Create a GitHub Account

If you don't already have one

  1. Go to github.com
  2. Click "Sign Up"
  3. Use your Cornell email (you get free perks!)
  4. Choose a professional username (you'll use this forever)
Pro tip: Your GitHub profile is like a portfolio. Employers WILL look at it.

Join Our Organization

Send your user in slack to general to be added as a collaborator

Once invited, you'll receive an email or notification.

Accept the invitation to join Cornell-Data-Strategy

Install Git

macOS

Open Terminal and run:

xcode-select --install

Follow the prompts to install

Windows

Download installer from:

git-scm.com/download/win

Use all default options

Verify Git Installation

Open your terminal and run:

git --version

You should see something like:

git version 2.42.0
If you see a version number, you're good!

Configure Git

Tell Git who you are:

git config --global user.name "Your Name"
git config --global user.email "your.email@cornell.edu"

Verify your config:

git config --list

Install GitHub CLI

Download the installer for your OS from:

cli.github.com

Run the installer and follow the prompts.

Login to GitHub

Run this command:

gh auth login
  1. Select GitHub.com
  2. Select HTTPS
  3. Select Login with a web browser
  4. Copy the one-time code
  5. Press Enter to open GitHub in browser
  6. Paste the code and authorize
You're now authenticated!

Clone the Repository

Navigate to where you want to store the project:

cd ~/Desktop

Clone the repo:

git clone https://github.com/Cornell-Data-Strategy/NME-Workshop-1.git

Enter the project folder:

cd NME-Workshop-1

Open in Your IDE

VS Code (Recommended)

If you have VS Code installed:

code .
Don't have VS Code?
Download from code.visualstudio.com → Install → Restart terminal

Add Your Name!

Open README.md and add your name:

# NME-Workshop-1

## Contributors
- [Your Name](https://github.com/your-username)
Save the file! (Cmd+S on Mac, Ctrl+S on Windows)

Stage Your Changes

See what files changed:

git status

You should see README.md in red (modified but not staged).

Add it to staging:

git add README.md

Now git status shows it in green!

Commit Your Changes

Create a commit with a message:

git commit -m "Add [Your Name] to contributors"

Replace [Your Name] with your actual name!

Commit messages should be short and describe WHAT you changed.

Push to GitHub!

Send your changes to GitHub:

git push
That's it! Your changes are now on GitHub!

Go to the repo on GitHub and refresh to see your name!

Quick Reference - Git Commands

CommandWhat it does
git clone <url>Download a repository
git statusSee what's changed
git add <file>Stage changes for commit
git add .Stage ALL changes
git commit -m "msg"Save staged changes
git pushUpload commits to GitHub
git pullDownload latest changes

The Git Workflow

Edit File
Make changes
Stage
git add
Commit

Push

Remember: Edit → Stage → Commit → Push

Troubleshooting

"Permission denied"

gh auth login

Re-authenticate with GitHub.

"Failed to push - rejected"

git pull

Then try git push again.

"Command not found: git"

Reinstall Git and restart your terminal.
